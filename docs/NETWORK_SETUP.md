# Сеть: LAN, туннели, облако

## Локальная сеть (LAN)

Все устройства должны быть в одной подсети (один Wi‑Fi или офисная сеть). Сервер слушает `0.0.0.0:8000`. Клиенты подключаются к `http://<IP_сервера>:8000`. Убедитесь, что брандмауэр на ПК с сервером пропускает порт.

## Ngrok (временный публичный URL)

1. Установите [ngrok](https://ngrok.com/), авторизуйтесь.
2. Запустите API локально на порту 8000.
3. В отдельном окне: `ngrok http 8000`.
4. Используйте выданный HTTPS URL в настройках клиентов. Бесплатный tier меняет адрес при перезапуске.

## Облако (VPS / PaaS)

Подходит для работы без постоянно включённого домашнего ПК. Типичный сценарий: контейнер или web service с `uvicorn`, отдельная PostgreSQL, переменные окружения для секретов и `DATABASE_URL`. Учтите «засыпание» бесплатных тарифов и холодный старт.

## HTTPS

Для доступа из интернета используйте HTTPS (туннель или хостинг с сертификатом). Не передавайте пароли по открытому HTTP в публичных сетях.
