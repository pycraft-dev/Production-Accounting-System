# Автообновление клиентов

## Сервер

- Версия API задаётся переменной **`APP_VERSION`** (или по умолчанию в коде настроек).
- Каталог артефактов — **`UPDATES_ROOT`** (по умолчанию `../updates` относительно рабочего каталога при запуске uvicorn из `backend`).
- Файл **`manifest.json`** в этом каталоге описывает версии и имена файлов:

```json
{
  "min_supported_client": "1.0.0",
  "desktop": { "version": "1.0.1", "filename": "desktop/update_v1.0.1.zip" },
  "mobile": { "version": "1.0.1", "filename": "mobile/app-v1.0.1.apk" }
}
```

- Поле `filename` может быть `null`: тогда эндпоинты скачивания вернут 404, клиенты не будут обновляться.
- Эндпоинты: **`GET /api/version`**, **`GET /api/updates/desktop/latest`**, **`GET /api/updates/mobile/latest`**.

## Десктоп (Windows)

- Локальная версия: файл **`desktop/version.json`** (`{"version": "1.0.0", ...}`).
- При старте приложение запрашивает `/api/version`; если версия десктопа на сервере новее — скачивается zip и распаковывается в каталог приложения, затем процесс перезапускается (`os.execv`).
- Требуются права записи в каталог установки; антивирус может блокировать распаковку.

## Мобильный (Android)

- Локальная версия: **`mobile/version.json`**.
- При запуске через ~1 с проверяется версия; при наличии более новой показывается диалог. По согласию APK скачивается в каталог данных приложения; установка выполняется пользователем (открыть файл), с включённым разрешением «неизвестные источники» при необходимости.

## Сборка пакета для десктопа

Положите в zip обновляемые файлы с той же структурой относительно корня `desktop/` (например `src/...`, `main.py`), обновите `version` в манифесте и в `version.json` после выкладки.
